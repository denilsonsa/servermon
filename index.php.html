These pages demonstrate what servermon looks like and feels like. They were generated from anonymized real-life logs. Obviously, the real-world servermon page does not contain this message.

There are 4 servers being monitored here:

  • kirk, which is a worldwide visible server inside st subnetwork.
  • spock, which is not directly accessible, but can be reached via kirk.st.example.com/spock, which is an Apache URL in kirk that uses mod_proxy to redirect to spock.
  • luke, which is a worldwide visible server inside sw subnetwork.
  • vader, which is a worldwide visible server inside sw subnetwork.

In this example, the servers inside st network are marked as important and will be highlighted. The servers on sw network are from a neighbor lab, and are not important. Since both st and sw share the same example.com network link, monitoring all those servers at once are a good indicator of failure outside the st network.

You can also download the sample server_list.conf used in this setup.
